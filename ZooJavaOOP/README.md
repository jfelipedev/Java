# Avaliação (Checkpoint)

## Objetivo

O objetivo deste checkpoint é utilizar conceitos de orientação a objetos apresentados 
durante o bimestre. Você deve implementar as classes modeladas no diagrama de 
classes e enviar todo o código fonte do sistema. 

## Requisitos 

### 1. Características básicas. 
- Ser orientado a objetos. 
- Usar o console para receber as entradas do usuário. 
- Todas as classes modeladas no diagrama de classes devem estar 
implementadas no código fonte.

### 2. Tópicos da disciplina que deverão ser contemplados e que serão avaliados no projeto: 
- Classes (atributos e métodos), Construtores, Encapsulamento.
- Implementar os conceitos de sobrecarga/sobrescrita (de métodos e 
construtores); 
- Implementar o conceito de herança, classe abstrata e/ou interface;
- Implementar os conceitos de Collections (List ou Set);
- Implementar os conceitos de Exceptions (RuntimeException e 
personalizadas).

## Proposta Fictícia 
A Secretaria de Cultura da cidade de Campina Grande-PB está visando ampliar o fluxo de turismo e lazer no município e uma das propostas é iniciar a construção de um zoológico.

Para isso, a ideia foi passada para um grupo de desenvolvedores com o propósito de iniciar uma das demandas, por ora subjetiva, dessa empreitada: realizar um levantamento cadastral de funcionários e animais.

Nesse primeiro estágio da aplicação, os desenvolvedores realizaram um levantamento mais objetivo, com intuito de avaliar de maneira quantitativa, ou seja, descobrir o número de funcionários e animais que o zoológico comportaria.

O quadro de funcionários, por ora, é mais simplificado, constando apenas dados pessoais e de moradia, já que a contratação de especialistas será terceirizada. Por outro lado, o quadro de animais foi dividido com base nas classes (mamíferos, peixes, anfíbios, répteis e aves), uma vez que cada classe de animal vai precisar de um espaço criado de maneira personalizada.
